# Test Plan - Proyek SQA Kelompok

## 1. Tujuan
Memastikan perangkat lunak berfungsi sesuai dengan kebutuhan pengguna dan bebas dari defect kritis.

## 2. Ruang Lingkup (Scope)
### Fitur yang Diuji
- Modul Login
- Modul Input Transaksi
- Modul Dashboard

### Fitur yang Tidak Diuji
- Modul Laporan (belum tersedia)

## 3. Jadwal Pengujian
| Aktivitas | Tanggal |
|-----------|--------|
| Persiapan Test Case | 1 April 2026 |
| Eksekusi Pengujian | 2-4 April 2026 |
| Pelaporan Bug | 5 April 2026 |

## 4. Kriteria Kelulusan
- Pass rate minimal 95%
- Tidak ada bug dengan severity High
